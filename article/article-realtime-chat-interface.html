Building a Real-Time AI Chat Interface
with JavaScript and Python APIs

Nobody wants to stare at a spinner for eight seconds. Here's how to build a chat interface that streams AI responses token by token — the same way ChatGPT and K-xpertAI do it.

Introduction

Why Is This Topic Important?

I remember the first AI chat app I built. You'd type your message, hit enter, and then just wait. The whole response would come back at once after five or six seconds of complete silence. It felt clunky. More than that, it felt like the app was broken even when it wasn't. Users don't actually mind waiting for an answer — what they hate is uncertainty. Not knowing if something is happening.

Streaming fixes this completely. Instead of buffering the entire response and dumping it on screen all at once, you send each token — each little piece of text — to the browser the moment the model generates it. The user sees the response building word by word, just like someone typing in front of them. Suddenly the same 5-second wait feels fine, because they can already see the beginning of their answer.

This is exactly how K-xpertAI works under the hood. The frontend opens a connection to the backend, the backend starts generating from the model, and every token gets pushed to the browser as soon as it exists. The whole thing is surprisingly simple once you understand Server-Sent Events — which is what we're building today.

We're going to build this in two parts: a Python FastAPI backend that streams responses using SSE (Server-Sent Events), and a vanilla JavaScript frontend that listens to that stream and renders each token as it arrives. No React, no heavy frameworks — just clean fundamentals you can drop into any project.

What Libraries Do You Need?

The backend is pure Python. No heavy ML frameworks needed here — we're just building the server layer that sits between the frontend and whatever AI model you're using. If you're pointing this at your own local model, you'll need a couple more packages, but for a clean API-connected backend, this is all you need:

TERMINAL — Python (backend)
$ pip install fastapi uvicorn python-dotenv httpx

For the frontend, we're using zero libraries. Vanilla JavaScript has everything we need — the fetch API with ReadableStream is built into every modern browser and handles streaming beautifully.

Backend packages

The Core

Building It — Step by Step

STEP 01 Set Up the FastAPI App and CORS

First, the skeleton. We need FastAPI running with CORS enabled, otherwise your browser will block requests from the frontend. This is the most common mistake beginners make, so let's get it right from the start.

Python — main.py 
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import os
from dotenv import load_dotenv

load_dotenv()  # reads your .env file for API keys

app = FastAPI(title="KX Chat API")

# Allow all origins in dev — tighten this in production
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

class ChatMessage(BaseModel):
    message: str
    history: list = []  # conversation history for multi-turn chat

STEP 02 Write the Streaming Generator

This is the heart of the whole system. We create an async generator function — a Python function that yields values instead of returning them all at once. Every time it yields a chunk of text, FastAPI immediately flushes it to the browser. That's how the streaming works.

Python — main.py (continued) 
import httpx
import json

async def stream_ai_response(message: str, history: list):
    api_key = os.getenv("AI_API_KEY")

    messages = [
        {"role": "system", "content": "You are a helpful AI assistant."},
        *history,               # spread in prior conversation turns
        {"role": "user", "content": message}
    ]

    async with httpx.AsyncClient() as client:
        async with client.stream(
            "POST",
            "https://api.your-ai-provider.com/v1/chat",
            headers={
                "Authorization": f"Bearer {api_key}",
                "Content-Type": "application/json"
            },
            json={"messages": messages, "stream": True},
            timeout=60.0
        ) as response:
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    data = line[6:]
                    if data == "[DONE]":
                        yield "data: [DONE]\n\n"
                        return
                    try:
                        chunk = json.loads(data)
                        # Extract just the text token from the response delta
                        token = chunk["choices"][0]["delta"].get("content", "")
                        if token:
                            yield f"data: {json.dumps({'token': token})}\n\n"
                    except:
                        pass
  • The function is an async generator — it uses yield instead of return, so it can send multiple values over time.
  • httpx.AsyncClient lets us make non-blocking HTTP requests from within our async FastAPI handler.
  • Every chunk we yield must follow the SSE format: data: {...}\n\n. Those two newlines at the end are required — they signal the end of one event to the browser.
  • We send [DONE] as a sentinel value so the browser knows to stop listening.

STEP 03 Create the Streaming Endpoint

Now we connect the generator to an actual route. FastAPI's StreamingResponse handles all the SSE plumbing for us — we just need to tell it the right media type.

Python — main.py (route) 
@app.post("/chat/stream")
async def chat_stream(body: ChatMessage):
    return StreamingResponse(
        stream_ai_response(body.message, body.history),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "X-Accel-Buffering": "no",  # disables Nginx buffering
        }
    )

# Run with: uvicorn main:app --reload --port 8000
🔧

The X-Accel-Buffering: no header is easy to miss but critical if you ever deploy behind Nginx. Without it, Nginx will buffer your entire SSE stream and send it all at once — completely defeating the purpose of streaming.

STEP 04 Build the JavaScript Frontend

This is where it all comes together visually. We use the fetch API with a ReadableStream reader to consume the SSE stream. The key insight is that fetch can read the response body progressively — you don't have to wait for it to finish.

JavaScript — chat.js 
const API_URL = 'http://localhost:8000/chat/stream';
let conversationHistory = [];

async function sendMessage(userMessage) {
  // Add user message to conversation history
  conversationHistory.push({ role: 'user', content: userMessage });

  // Create a new message bubble for the AI response
  const bubble = createAIBubble();

  try {
    const response = await fetch(API_URL, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        message: userMessage,
        history: conversationHistory.slice(0, -1)  // all except current
      })
    });

    const reader = response.body.getReader();
    const decoder = new TextDecoder();
    let fullResponse = '';

    while (true) {
      const { done, value } = await reader.read();
      if (done) break;

      const chunk = decoder.decode(value, { stream: true });
      const lines = chunk.split('\n');

      for (const line of lines) {
        if (!line.startsWith('data: ')) continue;
        const data = line.slice(6).trim();
        if (data === '[DONE]') break;

        try {
          const parsed = JSON.parse(data);
          const token = parsed.token || '';
          fullResponse += token;
          bubble.textContent += token;  // render token to DOM immediately
          scrollToBottom();
        } catch { /* skip malformed chunks */ }
      }
    }

    // Save complete response to history for multi-turn context
    conversationHistory.push({ role: 'assistant', content: fullResponse });

  } catch (err) {
    bubble.textContent = 'Connection error. Is the server running?';
    bubble.classList.add('error');
  }
}
  • reader.read() — this is a loop that keeps pulling bytes from the open HTTP connection until it closes. Each call gives you the next available chunk.
  • TextDecoder with stream: true — this flag is important. It tells the decoder that more data might be coming, so it won't break on partially received multi-byte characters.
  • bubble.textContent += token — appending directly to the DOM is what makes the text appear to "type itself" word by word.
  • We maintain conversationHistory so the model has context for follow-up questions — otherwise every message would be treated as a new conversation.

STEP 05 Wire Up the UI

The last piece is the actual HTML interface and the helper functions. Keep it minimal — the streaming effect itself is the impressive part.

JavaScript — UI helpers 
function createAIBubble() {
  const bubble = document.createElement('div');
  bubble.className = 'message ai-message';
  bubble.textContent = '';
  document.getElementById('messages').appendChild(bubble);
  return bubble;
}

function scrollToBottom() {
  const container = document.getElementById('messages');
  container.scrollTop = container.scrollHeight;
}

// Hook up the send button
document.getElementById('send-btn').addEventListener('click', () => {
  const input = document.getElementById('user-input');
  const message = input.value.trim();
  if (!message) return;

  const userBubble = document.createElement('div');
  userBubble.className = 'message user-message';
  userBubble.textContent = message;
  document.getElementById('messages').appendChild(userBubble);

  input.value = '';
  sendMessage(message);
});

// Allow Enter key to send
document.getElementById('user-input').addEventListener('keydown', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    document.getElementById('send-btn').click();
  }
});

Start the backend with uvicorn main:app --reload --port 8000, open your HTML file in a browser, and you should see responses streaming in word by word. That's the full stack working end to end.
